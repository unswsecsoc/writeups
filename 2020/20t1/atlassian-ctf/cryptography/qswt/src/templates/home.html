Querystring Web Tokens

The successor to JSON Web Tokens

Querystring Web Tokens are an open, industry standard RFC 1337 method for representing claims securely between two parties.

Why QSWT

Safe

It is so safe, that not even a safe-cracker can crack it in 1000 years. Using the uniquely patented SHA1 algorithm we are able to guarantee 160 bits of safety. Since 160 bitcoins is worth 1.5 million US dollars (May 2020), we guarantee that an attacker which is worth less than this will not be able to break this standard. This also means that the standard should not be used to secure assets that are worth more than 1.5 million US dollars.

Secure

Using agile methodologies currently only known to a select few, we blended just the right amount of memory, computer chips and network packets to create this standard. We publicised the final source, but not the secret ingredient in order to increase the security of our product. A compromise of our standard can only arise if someone was able to kidnap our developers from our underground bunker and then subsequently proceed to torture the secret key out of them.

State of the Art

Open source and patented you say, that's what makes this standard state of the art. React.JS also used to be patented, but then Facebook decided that NPM had grown so fragmented and that it was depended on so much code that they unpatented it because it had become so bad. Our implementation is not fragmented at all since it only depends on the Python standard library.
